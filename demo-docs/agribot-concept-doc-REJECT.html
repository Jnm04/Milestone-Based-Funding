AgriBot — Automated Crop Monitoring System

Prepared by: AgriBot Technologies Ltd.  |  Date: March 2026
Milestone Submission: Functional Prototype with Aerial Imagery Analysis

1. Project Background

AgriBot is developing an automated crop monitoring system that uses drone-captured aerial imagery combined with machine learning to detect crop health anomalies. The long-term vision is to provide farmers with affordable, scalable access to precision agriculture insights previously only available to large agribusiness operators.

2. System Concept

The proposed system consists of three components: a drone fleet for aerial data capture, an onboard image processing module, and a cloud-based analytics dashboard. Drones would fly pre-programmed routes over agricultural fields and transmit imagery to the processing backend.

We plan to use multispectral sensors to capture data across visible and near-infrared bands, which will be used to compute vegetation indices including NDVI (Normalized Difference Vegetation Index). Anomaly detection will flag areas where index values fall below crop-type-specific thresholds.

3. Technical Approach

Our team has been researching convolutional neural network architectures suitable for aerial agricultural imagery. We have reviewed published datasets including the PlantVillage dataset and several Kaggle agricultural competitions. The architecture we intend to use is based on a ResNet-50 backbone fine-tuned for multi-class crop stress detection.

We are also exploring integration with existing drone platforms such as DJI Agras and Parrot Sequoia to avoid building custom hardware from scratch. This would significantly reduce time-to-market.

4. Current Status

The team has completed initial research and begun setting up the development environment. We have acquired access to two test fields through a partnership with a local farming cooperative in Brandenburg, Germany. Field trials are planned for April 2026 once weather conditions permit drone operations.

The backend infrastructure is in early design phase. We are evaluating cloud providers and have shortlisted AWS and Google Cloud for the image processing pipeline. No final decision has been made yet.
